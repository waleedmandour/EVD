Chapter 1

Getting Started

EVDx is a privacy-first electric vehicle diagnostic app for Android 7.0+ that turns your phone into a professional-grade OBD-II scan tool. Connect a Bluetooth LE or WiFi ELM327 adapter, and EVDx reads live telemetry, scans diagnostic trouble codes, tracks charging sessions, and generates PDF reports — all without sending a single byte to the cloud.

Version Latest ships critical fixes to the BLE command pipeline, real diagnostic scan logic, and the disconnect flow. If you are upgrading from earlierx or earlier, your existing vehicle profiles and maintenance history are preserved automatically.

System Requirements

Android 7.0 (API 24) or later. Bluetooth LE recommended (most modern adapters). 25 MB free storage. No internet connection required — the app works fully offline once installed.

Chapter 1 · 1

Installation

  1. Download the APK from the Latest release page on GitHub. The file is named evdx-Latest.apk and is approximately 25 MB.
  2. On your Android phone, open Settings → Security (or Privacy on newer Android versions) and enable "Install unknown apps" for your browser or file manager.
  3. Open the downloaded APK file from your notifications or file manager and tap Install. The install takes about 10 seconds.
  4. Tap Open when the install completes. The EVDx onboarding wizard starts automatically.
Upgrading from an earlier version?

If you have earlierx or Latest installed, the Latest APK will install over it without losing your vehicle profiles, maintenance history, or settings. The new release uses the same application ID (com.waleedmandour.evdx) but a new versionCode (8), so Android recognises it as an update.

Chapter 3

Connecting an OBD-II Adapter

EVDx supports three connection modes: Bluetooth LE (recommended), WiFi ELM327, and Demo (simulated data for trying the app without an adapter). Tap the Connect button at the top of any screen to open the connection overlay.

Chapter 3 · 1

Bluetooth LE Connection

Most modern OBD-II adapters use Bluetooth LE. EVDx auto-detects 10 adapter profiles and shows every BLE device in range — including unknown names — so you can pick yours even if it isn't on the official list.

  1. Plug your OBD-II adapter into the vehicle's diagnostic port (under the dashboard, driver side). The adapter's LED should light up.
  2. In EVDx, tap Connect → Bluetooth OBD. Scanning starts automatically and runs for 8 seconds.
  3. On Android 12+, you'll be prompted to grant BLUETOOTH_CONNECT and BLUETOOTH_SCAN permissions. On Android 11 and below, you'll be asked for ACCESS_FINE_LOCATION (required for BLE scan results).
  4. When the scan finishes, OBD-like devices are listed at the top (sorted by signal strength). Tap yours to connect.
  5. EVDx runs the full ELM327 initialization sequence, queries adapter identification (firmware, chipset, protocol), detects supported PIDs, and reads the VIN. This takes 5-10 seconds.
What's new

Three major improvements: (1) The BLE service now uses a serialized command queue — concurrent commands can no longer corrupt each other's responses. (2) Three new BLE profiles added: "vGate iCar Pro v3+ (FFF0)" for 2023+ iCar Pro units, "BOLUTEK / CSR Clone (FFF0/FFF2)" for asymmetric write/notify clones, and the existing FFE0 profile renamed for clarity. (3) A profile picker appears automatically when auto-detection fails — instead of the confusing "characteristics not found" error, you now get a list of all 13 profiles to try manually.

Chapter 3 · 1b

Profile Picker — "Characteristics Not Found"

If EVDx connects to your adapter but can't find the expected BLE GATT characteristics, you'll see a "characteristics not found" error. This happens when the adapter uses a non-standard BLE profile that doesn't match any of the 13 known profiles in EVDx's database.

In Latest, instead of showing a dead-end error, EVDx now shows a profile picker dialog automatically. The dialog lists all 13 known BLE profiles with their service UUIDs. Tap each profile to retry the connection with it — EVDx will attempt to connect using that profile's service/characteristic UUIDs.

Vgate iCar Pro v3+ (2023+)

If your Vgate iCar Pro was purchased in 2023 or later, it likely uses the FFF0 BLE profile instead of the classic FFE0. When the profile picker appears, tap "vGate iCar Pro v3+ (FFF0)" first — this is the profile designed for newer iCar Pro units. If that doesn't work, try "BOLUTEK / CSR Clone (FFF0/FFF2)" next, then the remaining profiles in order.

How to use the profile picker:

  1. Connect to your adapter as usual (Connect → Bluetooth OBD → tap your device).
  2. If the connection fails with "characteristics not found", the profile picker appears automatically below the device list.
  3. Read the error message — it tells you which profile was tried and failed.
  4. Tap a profile from the list to retry. EVDx disconnects, reconnects, and tries the new profile's UUIDs.
  5. If the new profile works, you're connected. If not, try the next profile.
  6. If none of the 13 profiles work, your adapter may use a completely non-standard BLE profile. Report it on GitHub with your adapter's brand, model, and purchase year so we can add support.
Why does this happen?

BLE adapters expose a GATT service table with specific UUIDs. Different adapter manufacturers use different UUID schemes — Vgate iCar Pro v1/v2 uses FFE0/FFE1, Vgate iCar Pro v3+ uses FFF0/FFF1, OBDLink uses 18F0/2AF0/2AF1, vLinker uses Nordic NUS (6e400001/002/003), etc. EVDx's auto-detection probes each known profile's UUIDs; if none match, it can't know which UUIDs to use. The profile picker lets you manually override the auto-detection.

Chapter 3 · 2

WiFi ELM327 Connection

Some adapters (notably OBDLink WiFi and certain ELM327 WiFi clones) expose a TCP socket instead of BLE. EVDx can talk to them over local HTTP.

  1. Connect your phone to the adapter's WiFi network (usually named OBDII, WiFi_OBDII, or similar — check your adapter's manual).
  2. In EVDx, tap Connect → WiFi OBD.
  3. Enter the adapter's IP address (default 192.168.0.10) and port (default 35000).
  4. Tap Connect. The same ELM327 initialization sequence runs as for BLE.
WiFi caveat

WiFi ELM327 adapters expose a raw TCP socket, not HTTP. EVDx uses an HTTP fallback that works with many adapters but not all. If WiFi mode fails to connect, try BLE — most adapters sold today support both.

Chapter 3 · 3

Demo Mode

Demo mode runs the built-in multi-brand driving/charging simulator. It produces realistic EV telemetry for BYD (LFP), Tesla (NCA), Nissan (NMC), Hyundai/Kia (NCM), VW Group (NMC), or a generic profile. Use it to explore the app before buying an adapter, or to train staff without occupying a vehicle.

To start demo mode: Connect → Demo, pick a brand and model, then tap Start Demo Mode. The simulator immediately starts pushing live data to the dashboard. In Latest, running the diagnostic scan in demo mode also injects two sample DTCs (P0A80 hybrid battery deterioration, P0C79 drive motor inverter performance) so you can explore the diagnostic UI without a real vehicle.

Chapter 4

Running a Diagnostic Scan

Once connected (BLE, WiFi, or Demo), open the Diagnostics tab and tap Run Full Scan. EVDx queries the vehicle's ECU for three categories of diagnostic trouble codes:

Mode Type What it returns
03 Stored DTCs Active fault codes currently stored in the ECU
07 Pending DTCs Faults detected but not yet confirmed (may clear themselves)
0A Permanent DTCs Cleared codes that the ECU still tracks per EPA regulation

Each code is enriched with a description, severity (low / medium / high / critical), common causes, and recommended actions from the built-in 500+ DTC database. Tap any code to expand the details.

What was fixed

In earlierx and Latest, the "Run Full Scan" button only animated a progress bar — it never actually queried the vehicle. This is now fixed: the button issues real Mode 03/07/0A reads via bleService.readDTCs() and merges the results with descriptions from the DTC database. The "Clear Codes" button also now sends a real Mode 04 command to the ECU (previously it only cleared local UI state, so the codes would reappear on the next scan).

Chapter 4 · 1

Clearing Codes

To clear codes from the vehicle's ECU: tap Clear Codes in the Diagnostics tab, then confirm in the dialog. EVDx sends a Mode 04 command, waits for the positive response, and clears the local DTC list. The MIL (check engine light) should also turn off if all critical codes were cleared.

Mode 04 strict match

Latest also fixes a false-positive bug in the clear-DTC success check. The previous code used response.includes('44') which would match any response containing the substring "44" anywhere — including multi-byte OBD responses with embedded 0x44 bytes. The fix uses a strict prefix match on the whitespace-stripped response, so success is only reported when the ECU actually returns the Mode 04 positive response.

Chapter 5

Live Data & Polling

While connected, EVDx polls the vehicle's ECU at 500ms intervals for standard SAE J1979 Mode 01 PIDs. The polling list is filtered to only PIDs the vehicle actually supports (detected via the 0100/0120/0140/... bitfield query during connect).

PID Name Formula
0DVehicle SpeedA km/h
0CMotor RPM(A×256+B)/4
05Coolant / Motor TempA−40 °C
0FIntake Air / Inverter TempA−40 °C
42Control Module Voltage(A×256+B)/1000 V
46Ambient Air TempA−40 °C
2FFuel Level (SOC proxy)A×100/255 %
61Driver Demand TorqueA−125 %
62Actual Motor TorqueA−125 %
A6Odometer(A×16777216+B×65536+C×256+D)/10 km

Live data flows to the Dashboard, Battery, Charging, and Live Data tabs. The Live Data tab lets you graph up to 6 parameters simultaneously and export the window as CSV.

Chapter 6

Session Logging

The Sessions tab lets you record a trip for later review. Tap Start Trip before driving and Stop Trip when finished. EVDx captures: distance, average speed, max speed, energy consumed, energy recovered via regen, trip duration, start/end SOC, and an eco-score breakdown.

What was fixed

In previous versions, the session log only captured data in demo mode — when connected to a real BLE/WiFi adapter, the "Start Trip" button did nothing because the simulator (which writes the log) was stopped. Latest fixes this: the OBD response parser now writes sampled entries to the session log whenever logging is active, with a 2-second-per-PID rate limit to prevent flooding at the 500ms polling rate. OBD protocol errors (NO DATA, UNABLE TO CONNECT, BUS ERROR) are also logged so you can diagnose adapter issues during a trip.

Chapter 7

Voice Assistant

EVDx includes a bilingual voice assistant with on-device speech recognition and text-to-speech. Tap the floating microphone icon (bottom right) to start speaking, or tap the sparkles icon to have the current page summarized aloud.

The assistant recognises 90+ command phrases across Modern Standard Arabic, Gulf Arabic, Egyptian Arabic, and English. Navigation commands ("dashboard", "battery", "diagnostics") switch tabs. Data queries ("battery level", "range", "motor temp") read the current value aloud with severity-aware prosody.

Chapter 8

PDF Reports

EVDx can generate three types of PDF report entirely on-device using jsPDF:

B
Battery Health Report

SOC, SOH, cell voltage analysis, thermal management status, capacity charts, and care recommendations. Generated from the Battery tab.

D
Diagnostic Scan Report

Full DTC list with descriptions, severities, causes, and recommended actions. Includes monitor readiness status. Generated from the Diagnostics tab after a scan.

Chapter 9

Maintenance Tracking

The Maintenance tab lets you log service records: tire rotation, brake inspection, coolant flush, cabin filter, battery health, software update, or other. Each entry captures date, odometer, cost (OMR), notes, and next-due odometer. The full history is sortable and exportable.

What was fixed

Service type labels were previously displaying as raw keys (e.g. items.tire_rotation) because the locale files only had camelCase keys (e.g. items.tireRotation). Latest adds the 7 snake_case keys to both English and Arabic locale files, so service types now display correctly as "Tire Rotation", "Brake Inspection", etc.

Chapter 10

Privacy & Data

EVDx is built with a privacy-first architecture. The AndroidManifest declares no INTERNET permission for cloud traffic — the only network access is for local WiFi ELM327 adapter connections on 192.168.x.x. There are no analytics SDKs, no crash reporting, no user accounts, and no server-side data storage.

All vehicle data, session logs, maintenance records, and settings are stored on-device using Zustand's persist middleware (backed by localStorage). To export everything as JSON: Settings → Data Export. To delete all data: Settings → Delete All Data.

Verifying the privacy claim

You can verify EVDx makes no outbound network connections by running it in Android Studio's Network Inspector, or by using a firewall app like NetGuard to block all traffic for EVDx — the app will continue to work identically. The source code is open at github.com/waleedmandour/EVD.

Chapter 11

Troubleshooting

!
No adapters found in Bluetooth scan

Ensure your adapter is plugged in and powered (LED on). Move within 2 meters of the adapter. On Android 11 and below, grant ACCESS_FINE_LOCATION (required for BLE scan results). Restart Bluetooth. Tap Scan Again.

!
"UNABLE TO CONNECT" during initialization

The ELM327 cannot establish a protocol with the ECU. This usually means the vehicle's ignition is off — turn it to ON (or start the engine). For EVs, the vehicle must be in READY mode. Some adapters need a few seconds after plugging in; wait 10 seconds and try again.

!
"NO DATA" responses

The ELM327 is connected but the ECU didn't respond to a specific PID. This is normal — not all PIDs are supported on all vehicles. EVDx's polling loop automatically filters to supported PIDs, so you should only see this for one-off queries (like VIN). If every response is NO DATA, the protocol may be wrong — try ATSP0 (auto-detect, the default).

!
VIN shows extra digits or wrong characters

This was a bug in earlierx and Latest where multi-frame VIN responses were parsed incorrectly — frame counter bytes (01, 02, 03) were being decoded as ASCII digits and polluting the VIN. Latest fixes this. If you still see issues, your adapter may be returning malformed multi-frame responses; try a different adapter or report the issue on GitHub.

!
Demo mode shows 12.5 A pack current instead of 125 A

This was a 10× calculation bug in the simulator — the previous formula multiplied kW by 1000 (treating kW as W) AND divided by 10 in the rounding step, so 125 A displayed as 12.5 A. Latest fixes this. Demo mode now shows realistic pack currents.

!
"Characteristics not found" when connecting

This means EVDx connected to the adapter via BLE but couldn't find the GATT characteristics it needs to read/write OBD data. In Latest, this triggers the profile picker automatically — see Chapter 3 · 1b for full instructions. If your adapter is a Vgate iCar Pro purchased in 2023 or later, select "vGate iCar Pro v3+ (FFF0)" from the profile picker. If none of the 13 profiles work, your adapter may use a completely non-standard BLE profile — report it on GitHub with your adapter's brand, model, and purchase year.

!
Adapter firmware update

EVDx cannot update adapter firmware directly — each manufacturer provides their own tool. Open the Device tab in EVDx and scroll to the "Firmware Update Information" section for the official update path for your adapter brand. Vgate iCar Pro has no public firmware tool — use the profile picker instead. OBDLink and vLinker adapters can be updated via their respective Android apps.
